Отчет по созданным базам данных

1.Таблица пользователей (CustomUser)

    Модель: CustomUser
    Назначение: Хранение информации о пользователях системы.
    Поля:
        id: Уникальный идентификатор пользователя.
        password: Поле для хранения пароля пользователя.
        last_login: Дата и время последнего входа пользователя.
        is_superuser: Булево поле, указывающее, является ли пользователь суперпользователем.
        username: Уникальное имя пользователя.
        first_name: Имя пользователя.
        last_name: Фамилия пользователя.
        email: Электронная почта пользователя.
        is_staff: Булево поле, указывающее, является ли пользователь сотрудником.
        is_active: Булево поле, указывающее, активен ли пользователь.
        date_joined: Дата и время создания пользователя.
        StatusID: Связь с таблицей статусов доступа пользователей (StatusDostupa), по умолчанию ID=2.
        telefon: Телефон пользователя (по умолчанию пустое).
        Primechanie: Примечание (по умолчанию пустое).
        tg_Chat_ID: ID чата, откуда пришел пользователь (по умолчанию Null).
        tg_User_ID: ID юзера, пришедшего из Telegram (по умолчанию Null).

Эти изменения позволяют расширить стандартную модель пользователя Django
дополнительными полями, специфичными для вашего проекта.

2. Таблица статусов доступа (StatusDostupa)

    Модель: StatusDostupa
    Назначение: Хранение статусов доступа пользователей.
    Поля:
        ID: Уникальный идентификатор статуса.
        Status: Текстовое поле, описание статуса.

3. Таблица типов товаров (Tip_Tovara)

    Модель: Tip_Tovara
    Назначение: Хранение типов товаров.
    Поля:
        ID: Уникальный идентификатор типа товара.
        Tip: Текстовое поле, описание типа товара.

4. Таблица категорий товаров (Kat_Tovara)

    Модель: Kat_Tovara
    Назначение: Хранение категорий товаров.
    Поля:
        ID: Уникальный идентификатор категории товара.
        Kategoriya: Текстовое поле, описание категории товара.

5. Таблица товаров (Tovar)

    Модель: Tovar
    Назначение: Хранение информации о товарах.
    Поля:
        ID: Уникальный идентификатор товара.
        Nazvanie: Название товара.
        Cena: Цена товара.
        Img_url: ссылка на Изображение товара.
        Reiting: Рейтинг товара.
        Opisanie: Описание товара.
        ID_TipTovara: Связь с таблицей типов товаров (Tip_Tovara).
        ID_KetegorTovara: Связь с таблицей категорий товаров (Kat_Tovara).

6. Таблица статусов заказов (StatusZakaza)

    Модель: StatusZakaza
    Назначение: Хранение статусов заказов.
    Поля:
        ID: Уникальный идентификатор статуса заказа.
        Status: Текстовое поле, описание статуса заказа.

7. Таблица заказов (Zakaz)

    Модель: Zakaz
    Назначение: Хранение информации о заказах.
   Поля:
        ID: AutoField, первичный ключ.
        ID_TipZakaza: CharField, максимальная длина - 255 символов. Хранит тип заказа.
        ID_User: ForeignKey, ссылка на модель пользователя (из настроек AUTH_USER_MODEL), удаление каскадное.
        ID_adres: ForeignKey, ссылка на модель адреса (Adresa), удаление - SET_NULL, значение может быть NULL.
        Kolichestvo_klon: PositiveIntegerField, по умолчанию - 1. Хранит количество клонов заказа.
        Primechanie: TextField, максимальная длина - 600 символов, по умолчанию пустое, может быть пустым. Хранит примечания к заказу.
        DataZakaza: DateTimeField, автоматически устанавливается текущая дата и время при создании записи.
        Peredano_v_bot: BooleanField, по умолчанию - False. Указывает, передан ли заказ в бота.
        Dostavka_nado: BooleanField, по умолчанию - True. Указывает, требуется ли доставка.
        Sobrano: BooleanField, по умолчанию - False. Указывает, собрано ли заказ.
        Data_sborki: DateTimeField, может быть NULL, по умолчанию - None. Хранит дату и время сборки заказа.
        Peredano_dostavka: BooleanField, по умолчанию - False. Указывает, передан ли заказ в доставку.
        Data_peredano_v_dostavku: DateTimeField, может быть NULL, по умолчанию - None. Хранит дату и время передачи в доставку.
        Zakaz_oplachen: BooleanField, по умолчанию - False. Указывает, оплачен ли заказ.
        Data_Zakaz_oplachen: DateTimeField, может быть NULL, по умолчанию - None. Хранит дату и время оплаты заказа.
        Zakaz_dostavlen: BooleanField, по умолчанию - False. Указывает, доставлен ли заказ.
        DataDostavki: DateTimeField, может быть NULL, по умолчанию - None. Хранит дату и время доставки заказа.
        Zakaz_Poluchen: BooleanField, по умолчанию - False. Указывает, получен ли заказ.
        Data_Zakaz_Poluchen: DateTimeField, может быть NULL, по умолчанию - None. Хранит дату и время получения заказа.
        Zakaz_zakryt: BooleanField, по умолчанию - False. Указывает, закрыт ли заказ.
        Data_Zakaz_zakryt: DateTimeField, может быть NULL, по умолчанию - None. Хранит дату и время закрытия заказа.


8. Таблица  товаров для заказов (Otgruzka).
    В ней содержатся все товары из всех заказов
    Модель Otgruzka:
        - ID: Уникальный идентификатор записи.
        - ID_Zakaz: Связь с таблицей Zakaz (id), каждая запись привязана к конкретному заказу.
        - tovar_id: Связь с таблицей Tovar, каждая запись содержит товар.
        - quantity: Количество товара.
        - cena: Цена товара на момент оформления заказа.

9. Таблица адресов для доставок (Adresa).
    Модель Adresa:
        - ID: Уникальный идентификатор записи.
        - ID_Zakaz: Информационное поле, содержит ID заказа (по умолчанию Null).
        - ID_User: Информационное поле, содержит ID пользователя (по умолчанию Null).
        - Gorod: Название города для доставки.
        - adres: Адрес доставки (по умолчанию пусто).
        - kontakt: Контактное лицо при доставке (по умолчанию пусто).
        - telefon: Телефон для связи при доставке.
        - adres_ediniy: Булевое поле, True если адрес используется для нескольких заказов (по умолчанию False).
        - Nazvanie_adresa: Название адреса доставки, например, название пункта выдачи.


10. Таблица отзывов (BaseOtziv)

    Модель: BaseOtziv
    Назначение: Хранение отзывов пользователей о товарах.
    Поля:
        ID: Уникальный идентификатор отзыва.
        ID_User: Связь с таблицей пользователей (Users).
        ID_Tovar: Связь с таблицей товаров (Tovar).
        Otziv: Текст отзыва.
        ReitingTovara: Рейтинг товара (от 1 до 5).

11. Таблица отчетов (Otchet)

    Модель: Otchet
    Назначение: Хранение аналитических данных и отчетов по заказам.
    Поля:
        ID: Уникальный идентификатор отчета.
        Data: Дата отчета.
        ID_Zakaz: Связь с таблицей заказов (Zakaz).
        Itogo: Итоговая сумма по продажам.
        Rashod: Сумма по расходам.
        Dohod: Сумма по прибыли.
        Retab: Рентабельность (с двумя знаками после запятой).

12. Таблица элементов корзины (CartItem)

    Модель: CartItem
    Назначение: Хранение информации о товарах, добавленных в корзину.
    Поля:
        ID: Уникальный идентификатор элемента корзины.
        tovar: Связь с таблицей товаров (Tovar).
        quantity: Количество товара.

13. Модель Zhurnal_status_Zakaza используется для хранения записей о изменениях статуса заказов.

    Поля модели:
        ID: AutoField, первичный ключ.
        ID_Zakaza: ForeignKey, ссылка на модель заказа (Zakaz), удаление каскадное.
        Izmenenie: CharField, максимальная длина - 255 символов. Хранит описание изменения.
        pole_izm: CharField, максимальная длина - 255 символов, может быть пустым или NULL. Хранит поле, которое было изменено.
        json_str: CharField, максимальная длина - 600 символов. Хранит JSON строку с дополнительной информацией об изменении.
        Date: DateTimeField, может быть NULL, по умолчанию - None. Хранит дату и время изменения.
        peredano: BooleanField, по умолчанию - False. Указывает, передано ли изменение в чат бот Telegram.
